## 系统调用基础

### 什么是系统调用
系统调用是用户程序与操作系统内核交互的接口。从形式上看是 C 函数，但实际实现是通过汇编语言直接与内核通信。

### 基本系统调用示例

#### `fork()` 系统调用
```c
int pid = fork();
if (pid == 0) {
    // 子进程代码
} else {
    // 父进程代码
}
```

- `fork()` 创建当前进程的完全副本
- 返回值：在子进程中返回 0，在父进程中返回子进程的 PID
- 子进程和父进程拥有相同的内存内容，但是独立的地址空间

#### `exec()` 系统调用
```c
char *argv[] = {"echo", "hello", "world", 0};
exec("echo", argv);
```

- `exec()` 用新程序替换当前进程的内存映像
- 参数数组必须以 `NULL`（或 `0`）结尾，用于标识数组结束
- 在 C 语言中，字符串以 `\0` 结尾，`NULL` 在内存中的值就是 `\x00`

#### `wait()` 系统调用
```c
wait(0);
```

- 父进程等待子进程结束
- 防止子进程成为僵尸进程
- 父进程必须为子进程"收尸"

## 文件描述符和 I/O 重定向

### 文件描述符概念
- **文件描述符 0**：标准输入（stdin）
- **文件描述符 1**：标准输出（stdout），通常对应终端显示器
- **文件描述符 2**：标准错误（stderr）

### I/O 重定向实现原理

```c
// 重定向输出到文件的实现
close(1);                    // 关闭标准输出
open("output.txt", O_WRONLY | O_CREAT); // 打开文件，获得文件描述符 1
```

**工作原理**：
1. `close(1)` 关闭标准输出
2. `open()` 总是返回最小可用的文件描述符，由于 1 刚被释放，新文件获得描述符 1
3. 之后所有写入文件描述符 1 的数据都会重定向到该文件

### Shell 重定向示例
```bash
echo hello > output.txt
```

这个命令的实现过程：
1. Shell 执行 `fork()` 创建子进程
2. 子进程中执行 I/O 重定向操作
3. 子进程调用 `exec("echo", ["echo", "hello", 0])`
4. 父进程调用 `wait()` 等待子进程完成

## 进程管理

### Fork-Exec 模式的优势

为什么不直接提供一个 `spawn()` 函数而要分离 `fork()` 和 `exec()`？

**答案**：分离设计提供了灵活性，允许在调用 `exec()` 之前修改进程状态，例如：
- 设置 I/O 重定向
- 修改环境变量
- 关闭不需要的文件描述符

### 进程执行顺序

父子进程的执行顺序是不确定的，操作系统的调度器决定哪个进程先运行。但通过 `wait()` 可以实现同步。

```c
int pid = fork();
if (pid == 0) {
    printf("child\n");
} else {
    wait(0);  // 等待子进程结束
    printf("parent\n");
}
```

## xv6 操作系统环境

### 环境组成
- **xv6**：简化的 Unix 操作系统
- **RISC-V**：指令集架构（相当于硬件规范）
- **QEMU**：硬件模拟器，模拟 RISC-V 环境

### xv6 特点
- 是 Unix 的简化版本
- 专为教学设计
- 实现了核心的操作系统功能，但比真实操作系统简单很多

### 基本命令
xv6 支持类似 Unix 的基本命令：
- `ls`：列出文件
- `cat`：显示文件内容
- `echo`：输出文本
- `grep`：文本搜索

## 文件系统和字节流

### 文件作为字节流
在 Unix/xv6 中，文件被视为字节流：
- 文件内容按字节顺序连续存储
- 读取时按字节流方式顺序读取
- 没有特殊的文件结构，由应用程序解释文件格式

### 字符串处理
C 语言中的字符串处理：
- 字符串以 `\0`（null terminator）结尾
- `\0` 在内存中的值是 `\x00`
- 用于标识字符串或数组的结束位置

## 编程语言差异

### C 语言 vs Python
- **C 语言**：编译型语言，系统调用直接与操作系统内核交互
- **Python**：解释型语言，通过解释器实现系统功能，解释器内部使用系统调用

两种语言在操作系统层面最终都会使用相同的系统调用，但调用路径不同。

## 重要概念总结

1. **系统调用**：用户程序与内核交互的接口
2. **进程**：程序的执行实例，拥有独立的地址空间
3. **文件描述符**：操作系统中文件的抽象表示
4. **I/O 重定向**：改变程序输入输出目标的机制
5. **字节流**：将文件视为连续字节序列的抽象
6. **Fork-Exec 模式**：Unix 进程创建的经典模式

这些概念构成了理解操作系统工作原理的基础，后续课程将深入探讨每个主题的实现细节。
